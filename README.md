# ai-summary-challenge

Стенд для тестового задания: карточка компании с блоком «ИИ-описание», который долго
грузится, иногда пропадает и порождает лавину лишних запросов — к базе и к платной
языковой модели.

Задание — в **[TASK.md](TASK.md)**.

## Быстрый старт

Нужны: Docker с Compose v2, `make` и `python3` на хосте (последний — только для `make bench`).

```bash
make up
open http://localhost:3000/company/7707410283
```

Первая сборка занимает несколько минут. Дальше:

```bash
make bench    # измерить, во что обходится одно открытие карточки
make stats    # счётчики: вызовы LLM, запросы в БД
make logs     # логи api / web / llm-mock
make reset    # обнулить счётчики и забыть сгенерированное описание
make down     # погасить и удалить тома
```

Стенд занимает порты **3000, 8000, 8090, 5434, 6380**. Если какой-то из них уже занят —
поменяйте его в `docker-compose.yml`; на поведение сервиса это не влияет.

### Настоящий LLM-провайдер вместо мока

Бэкенд ходит в модель через обычный OpenAI SDK, так что мок подменяется тремя переменными
в `docker-compose.yml` (`LLM_BASE_URL`, `LLM_API_KEY`, `LLM_MODEL`) — код трогать не нужно.

Но задание рассчитано на мок: он один даёт воспроизводимую латентность, честный
rate-limit и счётчик вызовов, по которому измеряется результат. С быстрой боевой моделью
часть симптомов просто не проявится.

## Из чего собран стенд

```
┌─────────┐   /api/*   ┌──────────┐   /v1/*   ┌─────────┐   OpenAI API   ┌──────────┐
│ браузер │ ─────────▶ │   web    │ ────────▶ │   api   │ ─────────────▶ │ llm-mock │
└─────────┘            │ Next.js  │           │ FastAPI │                └──────────┘
                       │ (прокси) │           └────┬────┘                 5-8 с/вызов
                       └──────────┘                │                      16 rpm лимит
                                              ┌────┴────┐
                                              │ db      │  Postgres 16
                                              │ redis   │  Redis 7
                                              └─────────┘
```

Браузер никогда не ходит в бэкенд напрямую — только через серверные роуты Next.js
под `/api/*`. Это осознанное ограничение архитектуры, а не случайность.

`llm-mock` — подделка под провайдера LLM, но с настоящей латентностью, настоящим
rate-limit'ом бесплатного тарифа и счётчиком вызовов. Он же — измерительный прибор:
`GET /admin/stats` показывает, сколько раз приложение на самом деле сходило в модель.

| Порт | Сервис |
|---|---|
| 3000 | web (Next.js 16, React 19, React Query) |
| 8000 | api (FastAPI, SQLAlchemy 2, Redis) |
| 8090 | llm-mock |
| 5434 | Postgres |
| 6380 | Redis |
